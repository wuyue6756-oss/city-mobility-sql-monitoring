# 表结构设计

## 1. dim_zone（区域维表）
用于存储区域编号与区域名称映射关系。

字段：
- LocationID：区域编号
- Borough：行政区
- Zone：区域名称
- service_zone：服务区域类型

## 2. fact_trip（订单事实表）
用于存储出租车订单明细数据。

核心字段：
- tpep_pickup_datetime：上车时间
- tpep_dropoff_datetime：下车时间
- passenger_count：乘客数
- trip_distance：行程距离
- PULocationID：上车区域编号
- DOLocationID：下车区域编号
- fare_amount：基础车费
- tip_amount：小费
- total_amount：订单总金额
- payment_type：支付方式

## 3. vw_trip_clean（清洗视图）
在 fact_trip 基础上构建清洗视图，
主要用于：
- 剔除距离小于等于 0 的记录
- 剔除金额异常记录
- 剔除区域为空记录
- 剔除时间为空记录
- 限定分析时间范围为 2025-11-01 至 2025-11-30
