# city-mobility-sql-monitoring
基于城市出行数据的经营监控与异常预警 SQL 项目，包含指标体系、数据清洗、监控查询与结果展示。
## 1. 项目背景
本项目基于公开城市出行交易数据，模拟企业经营分析场景，构建经营监控指标体系，并进行异常波动识别。

## 2. 项目目标
- 统计订单量、总收入、客单价等核心指标
- 分析分时段、分区域经营表现
- 识别异常波动日期与异常区域
- 输出经营分析结果与项目文档

## 3. 数据文件
- ‘taxi_zone_lookup.csv’：区域维表
- ‘yellow_tripdata_2025-11_sample.csv’：订单样本数据

## 4. 项目结构
- docs/：项目说明文档
- data/：样本数据与数据来源说明
- sql/：建表、清洗、指标统计、异常分析 SQL
- outputs/：结果截图
- report/：项目汇报材料

## 5. 技术栈

- Gitee

## 7. 核心结果
### 7.1 每日经营指标
基于清洗后的订单数据，按天统计订单量、总收入、客单价和平均行程距离，
得到 2025-11 月份样本期间的经营变化趋势。
### 7.2 高收入日期识别
对每日收入进行排序，识别出收入较高的重点日期，
为后续波动解释和异常观察提供依据。
### 7.3 异常日期分析
基于每日收入计算 z-score，对异常程度较高的日期进行排序识别，
结果显示 2025-11-20、2025-11-21 等日期收入水平明显高于样本均值。
### 7.4 区域热度分析
按上车区域聚合后，JFK Airport、LaGuardia Airport、Midtown Center 等区域
在收入和订单量上表现较为突出，具有较高经营热度。
