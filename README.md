# city-mobility-sql-monitoring
基于城市出行数据的经营监控与异常预警 SQL 项目，包含指标体系、数据清洗、监控查询与结果展示。
## 1. 项目背景
本项目基于公开城市出租车订单数据，模拟企业经营分析场景，围绕订单量、总收入、客单价、区域热度与异常日期识别等问题，构建一个完整的 SQL 分析项目。

## 2. 项目目标
- 完成订单数据与区域维表的导入与整理
- 构建清洗视图，统一分析口径
- 统计每日订单量、每日总收入、客单价等核心经营指标
- 基于 Z-score 识别异常日期
- 结合区域维表分析上车区域收入 Top10
- 输出结果图表、项目文档和仓库作品集

## 3. 数据文件
- ‘taxi_zone_lookup.csv’：区域维表
- ‘yellow_tripdata_2025-11_sample.csv’：订单样本数据

## 4. 项目结构
- docs/：项目说明文档
- data/：样本数据与数据来源说明
- sql/：建表、清洗、指标统计、异常分析 SQL
- outputs/：结果截图
- report/：项目汇报材料

## 5. 技术栈
- SQL
- Excel
- Python（仅用于 parquet 转 csv）
- Gitee

## 6. 核心分析内容
- 数据导入：将订单样本数据与区域维表导入 MySQL
- 数据清洗：构建 vw_trip_clean 视图，过滤异常值并统一分析时间范围
- 核心指标分析：按天统计订单量、总收入、客单价、平均行程距离
- 异常日期识别：基于每日收入均值与标准差计算 Z-score，并识别异常程度较高的日期
- 区域收入分析：关联区域维表，统计上车区域收入 Top10

## 7. 核心结果
### 7.1 每日订单量与收入趋势
11 月样本期间订单量整体在约 900 至 1800 单之间波动，每日总收入整体在约 2.2 万至 5.3 万之间波动。11 月 20 日和 11 月 21 日订单量与收入均处于较高水平。
### 7.2 异常日期识别（Z-score）
基于每日收入计算 Z-score 后，2025-11-20、2025-11-21、2025-11-06、2025-11-02、2025-11-13
在样本期间表现出更高的异常程度，属于值得重点关注的日期。
### 7.3 区域收入分析
JFK Airport 和 LaGuardia Airport 的收入显著高于其他区域，说明机场场景对总体收入贡献较强；
与此同时，Midtown Center、Upper East Side South 等曼哈顿核心区域也具有较高经营热度。
